html을 움직이고 싶다

html로만 된 웹 문서에서 사용자가 할 수 있는 행동은 그것을 읽는 일 뿐이다. 인터넷으로 어디서든 접속할 수 있다는 것, 링크로 연결하고 이동하는 기능은 획기적이었지만 그것이 컴퓨터와 인터넷의 한계일 리가. 정적인 웹 문서를 동적으로 만들기 위해 자바스크립트(이하 js)가 출현한다.

수식을 입력하면 계산 결과를 보여주고, 버튼을 누르면 안내창이 뜨고, 마우스를 올리면 스타일이 바뀌며 내용이 강조되고, 검색어를 입력하면 그것이 들어간 페이지를 찾아 표시해준다. 물론 쉽지는 않다. 사람들은 js가 온갖 일을 수행할 수 있기를 원했고, 결국 그 명령을 내리는 방법은 복잡해졌다.

html 기초 후 바로 배우는 건 (내가) 힘들었다. html은 인터넷 세상에 문서를 데려다 놓는 일이고, css는 옷을 입히는 일이라 생각한다. 날아다니는 문서에 옷을 입히는 건 어려울 뿐더러 순서가 맞지 않는다. 게다가 옷 입기는 훨씬 쉽다.

js 문법으로 화면에 글쓰기

js라는 언어를 배우기로 했다면 가장 기초적인 용도로 쉬운 말부터 해보자. 그것은 물론 html을 바꿔서 hello world 말하기. 할 일이 정보 전달뿐인 html은 우리가 쓴 말을 그대로 화면에 띄우지만, js에게는 '어디에 어떤 것을 어떻게 해줘' 라고 짚어줘야 한다. 이 경우엔 js야 '문서에 hello world를 적어줘' 가 되어야겠지.

'js야'

html에서 css를 부르는 style 태그가 있듯이, js는 script 태그로 부를 수 있다.
